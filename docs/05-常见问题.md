# 05 — 常见问题

> 按"症状"查。看完没解决再发 issue。

---

## Bot 不回话

### 第一步：确认 listener 在跑

```bash
launchctl print "gui/$(id -u)/ai.lark-inspiration-bot.listener" 2>/dev/null | grep state
```

输出应该是 `state = running`。

如果是 `not running` 或没输出：

```bash
./scripts/reload.sh
```

### 第二步：看日志

```bash
tail -f logs/listener.log
```

发条飞书消息，正常应该看到：

```
[xxx] [listener] 收到消息：你刚发的内容
[xxx] [listener] enqueue (first=1): ...
[xxx] [listener] 启动 flusher (pid=xxx)
```

如果**完全没输出**：listener 没收到事件，看 `logs/listener.err.log`。

### 第三步：常见原因

| 症状 | 原因 | 解法 |
|------|------|------|
| listener.log 一直没新内容 | 飞书事件订阅没配 | 重做 [docs/01](01-申请飞书机器人.md) 第五步 |
| `忽略非马克消息` 类似日志 | open_id 填错了 | 重新拿 open_id，改 `.env`，reload |
| `LLM API error` | DashScope key 无效 / 余额不足 | 看 `logs/llm.err.log`，去阿里云充值 |
| `lark-cli: command not found` | launchd 找不到 lark-cli | 见下面"PATH 问题" |

## PATH 问题（launchd 找不到 lark-cli）

launchd 启动的进程用的是**最小 PATH**，找不到你 shell 里的命令。

`templates/launchd-listener.plist.tpl` 里已经预设了常见路径：

```
/usr/local/bin:/opt/homebrew/bin:~/.lark-cli/bin:/usr/bin:/bin
```

如果你的 lark-cli 装在别的地方（比如 nvm、conda）：

```bash
# 找出 lark-cli 在哪
which lark-cli

# 拿到目录路径，比如 /Users/xxx/.nvm/versions/node/v20.0.0/bin/lark-cli
# 把那个目录加到 plist 模板里的 PATH
vim templates/launchd-listener.plist.tpl

# 重新跑 install
./install.sh
```

## 飞书发图片/语音/文件，bot 不理

设计如此。当前版本只支持文本消息。

如果你想加图片支持，得改 `scripts/listener.sh` 的 `msg_type` 过滤逻辑，并在 `reply.sh` 里处理图片下载。属于二次开发范围。

## Markdown 文件没写

```bash
# 看 reply 的日志
tail -f logs/reply.log

# 看 LLM 错误
tail -f logs/llm.err.log
```

常见原因：

| 症状 | 原因 |
|------|------|
| `LLM 返回空` | API key 失效 / 余额不足 |
| `JSON 解析失败` | LLM 返回了不是 JSON 的内容（偶发，下次会自愈） |
| `[append] 拿锁超时` | 你磁盘可能挂了 |

## 改了 .env 没生效

每次改 .env 后必须 reload：

```bash
./scripts/reload.sh
```

不 reload 的话 launchd 还在用老的 .env。

```bash
launchctl bootout "gui/$(id -u)/ai.lark-inspiration-bot.listener"
```

想再开：

```bash
./scripts/reload.sh
```

## 同样的消息回复了多次

罕见，但发生过：listener 启了两个实例。

```bash
# 看下是不是有多个进程
ps aux | grep listener.sh | grep -v grep

# 如果有多个，杀光重启
pkill -f listener.sh
./scripts/reload.sh
```

## API 费用爆炸了

DashScope 后台 https://bailian.console.aliyun.com/?tab=expense 看用量。

正常情况一天聊天 < 0.1 元。如果一天用了 1 元以上，可能是：

1. listener 起了多个实例（消息重复处理）→ 见上一条
2. 用了 qwen-max 而不是 qwen-plus → `.env` 改回 `LLM_MODEL=qwen-plus`
3. BOT_PERSONA 太长（每次调用都带它，长 = 贵）→ 精简到 200 字以内

