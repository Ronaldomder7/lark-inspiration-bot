# 01 — 申请飞书自建机器人

> 目标：拿到 `LARK_APP_ID` 和 `LARK_APP_SECRET`，并完成机器人到你飞书账号的授权。

---

## 第一步：去飞书开放平台创建应用

1. 打开 https://open.feishu.cn/app
2. 用你的飞书账号登录（个人版/企业版都行）
3. 点 **"创建企业自建应用"**
4. 填：
   - **应用名称**：随便填，比如 `灵感记录助手`
   - **应用描述**：随便填
   - **应用图标**：随便上传一张
5. 点创建

---

## 第二步：拿 App ID 和 App Secret

创建好后进入应用详情页：

1. 左侧菜单点 **"凭证与基础信息"**
2. 看到：
   - **App ID**：以 `cli_` 开头的一串字符
   - **App Secret**：点"显示"后能看到的一串字符

把这两个值复制下来，等会 `install.sh` 会问你。

---

## 第三步：开机器人能力

1. 左侧菜单点 **"添加应用能力"**
2. 找到 **"机器人"**，点开通
3. 开通后会出现一个 **机器人** 的菜单项

---

## 第四步：配置权限（关键，不配 bot 没法收发消息）

左侧菜单点 **"权限管理"**，搜索并开通这些权限：

### 必须开的：

| 权限 | 干啥用的 |
|------|---------|
| `im:message` | 收发消息 |
| `im:message:send_as_bot` | 用机器人身份发消息 |
| `im:resource` | 读消息内容 |
| `contact:user.id:readonly` | 通过 ID 找用户 |

### 怎么开：

每个权限旁边点"开通"，然后**右上角点"创建版本并发布"**，等管理员审批通过（个人飞书自建应用一般几分钟内秒过）。

---

## 第五步：开事件订阅（让 bot 能"听"到你发的消息）

1. 左侧菜单点 **"事件与回调"** → **"事件配置"**
2. **订阅方式**：选 **"使用长连接接收事件"**（不是 webhook，因为我们用 lark-cli 长连）
3. **订阅事件**：搜索并添加：
   - `im.message.receive_v1`（接收消息）
4. 保存

---

## 第六步：把 bot 加到你自己的飞书

1. 进入应用详情 → 左侧 **"应用发布"** → **"版本管理与发布"**
2. 点 **"创建版本"**，填个版本号（比如 `1.0.0`），描述随便，**可见范围选"仅自己"**
3. 提交审批（个人自建应用秒过）

发布通过后，打开飞书 app，搜索你刚才创建的应用名称，进入私聊，应该能看到 bot 了。

---

## 第七步：用 lark-cli 登录授权

回到终端：

```bash
# 先确认 lark-cli 装好了
lark-cli --version

# 走授权流程（会让你填 App ID / App Secret，然后开浏览器扫码）
lark-cli auth login --as bot
```

按提示填刚才拿到的 App ID 和 App Secret，然后在浏览器扫码授权完成。

---

## 验证

```bash
lark-cli im +messages-send --as bot --user-id <你的openid> --text "hello from bot"
```

如果飞书里收到这条消息，说明 bot 通了。`<你的openid>` 怎么拿，看 [docs/03-找到自己的-open-id.md](03-找到自己的-open-id.md)。

---

## 常见坑

- **创建版本时审批一直没过**：把可见范围改成"仅自己"，会自动通过。
- **找不到"事件与回调"菜单**：先在"添加应用能力"里把"机器人"开通。
- **lark-cli auth login 报错**：检查 App ID / Secret 是不是复制错了，前后没有空格。
